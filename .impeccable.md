## Design Context

### Users

MicroLearn is used by students who want short practical courses and by teachers who publish and manage learning content. The main jobs are discovering courses, enrolling, tracking learning progress, and managing course materials.

### Brand Personality

Editorial, structured, confident. The interface should feel like an educational magazine: strong typography, clear hierarchy, precise spacing, and a serious but approachable learning tone.

### Aesthetic Direction

Keep the existing newspaper/editorial visual language: large display headlines, compact uppercase labels, rule lines, asymmetric grids, restrained color accents, and light-mode-first presentation. Avoid generic SaaS cards, decorative gradients, and overly playful visuals.

### Design Principles

- Use typography and spacing as the primary visual system.
- Keep interactions explicit: every action should end with a clear success, loading, empty, or error state.
- Preserve the current editorial grid and rule-line language instead of replacing it with generic dashboards.
- Treat the project as a production-like academic LMS: polished, understandable, and demo-stable.
